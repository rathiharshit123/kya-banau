# Kya Banau? 🍛

> "Aaj kya banau?" — AI-powered Indian meal suggestions personalised to your family's taste, cuisine, and health goals.

## Stack

| Layer | Tech |
|---|---|
| Frontend + Backend | Next.js 15 (App Router, TypeScript) |
| UI | Tailwind CSS v4 + shadcn-style components |
| Database | Supabase Postgres (service-role, no RLS) |
| AI | OpenAI `gpt-4o-mini` with Structured Outputs |
| Recipe videos | YouTube Data API v3 (on-demand search) |
| Identity | Anonymous UUID device-id in localStorage |
| Validation | Zod on every API route |
| Deployment | Vercel |

## Getting Started

### 1. Clone and install

```bash
git clone <repo>
cd kya-banau
npm install
```

### 2. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and sign up / log in.
2. Click **New project**, pick an organisation, name, database password, and region (choose one close to your users, e.g. Mumbai/Singapore for India).
3. Wait for the project to finish provisioning (~1–2 min).

**Get your credentials:**

1. Open your project → **Project Settings** (gear icon) → **API**.
2. Copy **Project URL** → this is `SUPABASE_URL`.
3. Copy **service_role** key (under *Project API keys*) → this is `SUPABASE_SERVICE_ROLE_KEY`.

> Use the **service_role** key, not the anon key. This app has no user auth — the backend talks to Supabase directly.

**Run database migrations:**

In **SQL Editor** → **New query**, run each migration file in order:

1. `supabase/migrations/0001_init.sql` — households + suggestions tables
2. `supabase/migrations/0002_redesign.sql` — updated household schema
3. `supabase/migrations/0003_meal_interactions.sql` — click/interaction tracking for analytics

Or with the [Supabase CLI](https://supabase.com/docs/guides/cli):

```bash
npx supabase link --project-ref <your-project-ref>
npx supabase db push
```

### 3. Get API keys

#### OpenAI

1. Go to [platform.openai.com](https://platform.openai.com).
2. **API keys** → **Create new secret key**.
3. Copy the key → `OPENAI_API_KEY`.

#### YouTube Data API v3 (for recipe videos)

Required only if you want the **Watch Recipe Video** feature. Without it, meal suggestions still work — video search will fail silently on that button.

1. Go to [Google Cloud Console](https://console.cloud.google.com).
2. Create a project (or pick an existing one).
3. **APIs & Services** → **Library** → search **YouTube Data API v3** → **Enable**.
4. **APIs & Services** → **Credentials** → **Create credentials** → **API key**.
5. Copy the key → `YOUTUBE_API_KEY`.

Optional but recommended: restrict the key to **YouTube Data API v3** only (Credentials → your key → *API restrictions*).

Free tier gives ~10,000 quota units/day. Each recipe search uses ~100 units, so roughly ~100 video searches/day on the free tier.

### 4. Environment variables

```bash
cp .env.example .env.local
```

Fill in `.env.local`:

| Variable | Where to get it |
|---|---|
| `SUPABASE_URL` | Supabase → Settings → API → Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Settings → API → service_role key |
| `OPENAI_API_KEY` | [platform.openai.com](https://platform.openai.com) → API keys |
| `YOUTUBE_API_KEY` | [Google Cloud Console](https://console.cloud.google.com) → YouTube Data API v3 |

Example:

```env
SUPABASE_URL=https://xxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...
OPENAI_API_KEY=sk-...
YOUTUBE_API_KEY=AIza...
```

### 5. Run dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Features

- **Onboarding** — dietary type, cuisines, health goals, dislikes, kids, BMI
- **Meal generation** — pick scope (today / breakfast / lunch / dinner / week / tiffin) → AI suggests personalised Indian meals
- **Quick recipe** — step-by-step instructions generated by AI, expandable per dish
- **Watch Recipe Video** — on-demand YouTube search (`indian {dish} recipe home cooking`); only hits YouTube when the user clicks, never hallucinated by AI
- **WhatsApp share** — copy or share meal plan to family group
- **Analytics-ready** — generations, preferences, and meal interactions stored in Supabase for a future public dashboard

## Deploy to Vercel

1. Push to GitHub and import the repo in [Vercel](https://vercel.com).
2. Add all four environment variables in **Project Settings → Environment Variables**:
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `OPENAI_API_KEY`
   - `YOUTUBE_API_KEY`
3. Deploy — standard Next.js app, no extra config needed.

## How it works

- **No auth** — a UUID v4 is generated in `localStorage` on first visit and sent as `x-household-id` on every API call. This is the household identity.
- **Preferences** — stored in the `households` table when the user completes onboarding or updates preferences.
- **Suggestions** — every "Generate meal plan" call is logged in `suggestions` with the full AI response (greeting, tip, all meals).
- **Recipe videos** — when a user clicks **Watch Recipe Video**, the backend searches YouTube Data API v3 with Indian home-cooking context. Links are never generated by AI.
- **Interaction tracking** — expanding Quick Recipe, requesting videos, and clicking a YouTube link are logged in `meal_interactions` (for a future analytics dashboard).
- **Rate limit** — 20 AI suggestion requests per device per day.
- **Cost** — ~$0.0005 per suggestion call with gpt-4o-mini; YouTube searches are free within Google's daily quota.

## Data stored (for your dashboard)

| Table | What's in it |
|---|---|
| `households` | User preferences (diet, cuisines, health goals, dislikes…) |
| `suggestions` | Every AI generation — full meal plan JSON + meal scope + timestamp |
| `meal_interactions` | Clicks — `quick_recipe`, `recipe_videos`, `video_click` with dish name + metadata |

Example dashboard queries you'll be able to run:

- Most suggested / most clicked dishes
- Quick recipe vs video engagement rate
- Top YouTube channels/videos clicked
- Daily generations and active households
